Get Started in Seconds

Use our resolvers anywhere with these easy-to-follow guides.

Windows Setup

On Windows 11, you can set up DNS-over-HTTPS (DoH) system-wide for all applications. For other versions, you can configure it within your web browser.

  1. Navigate to chrome://settings/security
  2. Scroll down and enable "Use secure DNS".
  3. Select "With Custom" and enter a DoH URL from above (e.g., `https://freedom.mydns.network/dns-query`).
  1. Go to Settings → Privacy & Security.
  2. Scroll to the bottom and find "Enable DNS over HTTPS".
  3. Select "Max Protection", click "Choose a provider...", select "Custom", and enter a DoH URL.
  1. Navigate to edge://settings/privacy
  2. Scroll down to the Security section and enable "Use secure DNS".
  3. Select "Choose a service provider" and enter a DoH URL.
  1. Navigate to brave://settings/security
  2. Enable "Use secure DNS".
  3. Select "With Custom" and enter a DoH URL from above.

macOS Setup

On macOS Big Sur or newer, install a configuration profile for system-wide encrypted DNS. For a simpler setup, you can also configure it per-browser.

1. Select a Resolver

Note: macOS will show a warning during installation. This is expected because the profile is generated securely and dynamically in your browser.

2. Install the Profile

  1. Double-click the downloaded .mobileconfig file.
  2. System Settings will open. Click Install and follow the prompts.
  3. You can view installed profiles under System Settings → Privacy & Security → Profiles.

Linux Setup (systemd-resolved)

For modern Linux distributions using `systemd-resolved`, you can enable system-wide DNS-over-TLS.

  1. Edit the configuration file: sudo nano /etc/systemd/resolved.conf
  2. Uncomment and edit the following lines under the `[Resolve]` section:
DNS=# Optional fallback DNS
FallbackDNS=
Domains=~.
DNSOverTLS=yes
DNSSEC=yes
  1. Add one of our DoT hostnames to the `DNS=` line. For example: `DNS=freedom.mydns.network`
  2. Save the file and restart the service: sudo systemctl restart systemd-resolved

Android Setup

Android 9+ supports Private DNS, which uses DNS-over-TLS (DoT) to encrypt your queries. Later versions, like Android 11+, also add support for DNS-over-HTTPS (DoH), though DoT is the simplest to configure system-wide via the settings below.

  1. Go to Settings → Network & internet.
  2. Tap on Private DNS.
  3. Select Private DNS provider hostname.
  4. Enter a DoT address (e.g., `freedom.mydns.network`) and tap Save.

iOS Setup

On iOS 14+, you need to install a configuration profile to enable encrypted DNS system-wide. Use the tool below to generate a profile for your chosen resolver.

1. Select a Resolver

Note: iOS will show a "Not Verified" warning during installation. This is expected because the profile is generated securely and dynamically in your browser.

2. Install the Profile

  1. After downloading, open Settings and tap on Profile Downloaded.
  2. Tap Install in the top-right corner and enter your passcode.
  3. Tap Install again to confirm.

3. Activate the Profile

  1. Go to Settings → General → VPN & Device Management.
  2. Tap on DNS and select the newly installed profile.

JavaScript (Fetch API)

Use the Fetch API with modern async/await syntax in your web projects.

async function getDnsRecords() {
  const url = 'https://freedom.mydns.network/resolve?name=example.com';
  try {
    const response = await fetch(url);
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Error fetching DNS records:', error);
  }
}
